📋 System Overview

System Purpose

Windstorm-Dwarven is a FastAPI-based backend service that provides comprehensive API support for the DigitalForge ecosystem. It manages model-based systems engineering (MBSE) workflows, including requirements management, verification tracking, action orchestration, and artifact/container registry integration.

Key Capabilities

Capability Description Technology
Model Management Stores and tracks SysML v2 model commits, elements, and relationships PostgreSQL, SQLAlchemy
Requirements Traceability Manages requirements and their verification status across model versions Relational Data Model
Verification Orchestration Coordinates execution of verification actions with containers and artifacts Harbor Registry, Git Repositories
Artifact Storage Manages execution inputs/outputs via object storage MinIO S3-compatible storage
Authentication OAuth2/OIDC integration for secure API access Keycloak
RESTful API Comprehensive REST endpoints for all operations FastAPI, OpenAPI

System Characteristics

  • Python 3.11
  • FastAPI Framework
  • PostgreSQL Database
  • Docker Containerized
  • OAuth2 Secured
  • RESTful Design

Requirements Definition

System Requirements

  • REQ-001: System shall authenticate users via OAuth2/OIDC using Keycloak
  • REQ-002: System shall store and version SysML v2 model elements in PostgreSQL
  • REQ-003: System shall track requirements and their verification status
  • REQ-004: System shall resolve container images from Harbor registry
  • REQ-005: System shall resolve artifacts from Git repositories
  • REQ-006: System shall orchestrate thread executions with proper state management
  • REQ-007: System shall generate presigned URLs for artifact access via MinIO
  • REQ-008: System shall provide paginated API responses for all list operations
  • REQ-009: System shall support CORS for configured origins
  • REQ-010: System shall handle database connection failures gracefully
